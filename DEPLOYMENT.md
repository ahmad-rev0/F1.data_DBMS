# Deploy: Aiven (MySQL) + Render (Django) + Vercel (Next.js)

This is the recommended split: **Aiven** hosts MySQL (public hostname, free tier friendly for this dataset), **Render** runs the Dockerized Django app, **Vercel** serves the small `frontend/` app that calls `/api/…` on Render.

---

## 1. Aiven — MySQL

1. Sign in at [console.aiven.io](https://console.aiven.io/).
2. **Create service** → **MySQL**. Pick a region close to your Render region (lower latency).
3. Choose the **free** (or trial) plan if available; confirm **disk quota** vs your full import.
4. When the service is **Running**, open **Overview** and copy:
   - **Host**
   - **Port**
   - **User** (often `avnadmin`)
   - **Password** (service password from when you created it, or reset under Users)
   - **Database name** (often `defaultdb` unless you created another DB)

5. **TLS:** Aiven encrypts traffic. This project supports TLS via env vars (see below).  
   - If connections fail with certificate errors, download the **CA certificate** from the same Overview page (Aiven docs: *TLS/SSL certificates*) and either:
     - paste the PEM into Render as **`MYSQL_SSL_CA_PEM`** (multiline secret), or  
     - save the file as `deploy/aiven-ca.pem`, commit to a **private** repo if acceptable, set **`MYSQL_SSL_CA=/app/deploy/aiven-ca.pem`** on Render and ensure the file is copied in the Docker image (it is under `/app` by default).

6. **Allow connections from Render**  
   In Aiven: **Networking** (or service settings) → allow access from **everywhere** for a class project, or restrict to **Render outbound IPs** if you use static egress (advanced). For a first successful deploy, many teams temporarily use **0.0.0.0/0** on the MySQL service (understand the security tradeoff).

7. **Apply schema and load data** (from your laptop, using the same credentials):

   ```bash
   # .env — use your Aiven values
   MYSQL_HOST=mysql-xxxx.a.aivencloud.com
   MYSQL_PORT=12345
   MYSQL_DATABASE=defaultdb
   MYSQL_USER=avnadmin
   MYSQL_PASSWORD=...
   MYSQL_USE_SSL=true
   # Optional if Aiven requires project CA:
   # MYSQL_SSL_CA=D:\path\to\ca.pem

   mysql -h %MYSQL_HOST% -P %MYSQL_PORT% -u %MYSQL_USER% -p --ssl-mode=REQUIRED < database/schema.sql
   ```

   On Windows PowerShell you can use `Get-Content database/schema.sql | mysql ...` or a GUI (MySQL Workbench, DBeaver) pointed at Aiven with SSL.

   Then run optional SQL files (`indexes.sql`, `procedures.sql`, `triggers.sql`, `patch_lap_reporting_fact.sql`) the same way.

   Load CSVs from the repo root (with venv and `CSV files/` present):

   ```bash
   python scripts/load_f1_csv.py
   ```

## 2. Render — Django (Docker)

1. Push the repo to GitHub/GitLab.
2. **New** → **Blueprint** (or **Web Service** from the repo). If using Blueprint, connect this repo and apply **`render.yaml`**.
3. When prompted (or under **Environment** for `f1-platform`), set:

| Key | Value |
|-----|--------|
| `DJANGO_ALLOWED_HOSTS` | `your-service.onrender.com` (no `https://`) |
| `DJANGO_CSRF_TRUSTED_ORIGINS` | `https://your-service.onrender.com` |
| `DJANGO_CORS_ALLOWED_ORIGINS` | Your Vercel URLs, comma-separated (see §3) |
| `MYSQL_HOST` | Aiven host |
| `MYSQL_PORT` | Aiven port (string is fine) |
| `MYSQL_DATABASE` | e.g. `defaultdb` |
| `MYSQL_USER` | e.g. `avnadmin` |
| `MYSQL_PASSWORD` | Aiven password |
| `MYSQL_USE_SSL` | `true` (already in `render.yaml`; keep for Aiven) |
| `MYSQL_SSL_CA_PEM` | *Optional* — full CA PEM if required |
| `MYSQL_SSL_CA` | *Optional* — path inside container, e.g. `/app/deploy/aiven-ca.pem` |

`DJANGO_SECRET_KEY` can stay auto-generated by Render.

4. Deploy. On start, the container runs **migrations** then **Gunicorn**.
5. Smoke test: `https://<host>/api/health/` then `https://<host>/api/stats/` (stats need data in Aiven).

---

## 3. Vercel — `frontend/`

1. **Add New** → **Project** → import the same Git repo.
2. **Root Directory:** `frontend`
3. **Environment variables**
   - `NEXT_PUBLIC_API_URL` = `https://your-service.onrender.com` (no trailing slash)
4. Deploy.
5. Add your production (and preview) Vercel origins to **`DJANGO_CORS_ALLOWED_ORIGINS`** on Render, e.g.  
   `https://f1-frontend.vercel.app,https://f1-frontend-git-main-<team>.vercel.app`  
   Redeploy Render after changing CORS.

## Local `.env` (Aiven + optional TLS)

See `.env.example`. For local app + loader against Aiven, set the same `MYSQL_*` and `MYSQL_USE_SSL=true`.

---

## TLS environment variables (reference)

| Variable | Meaning |
|----------|--------|
| `MYSQL_USE_SSL` | Set to `true` for Aiven / any host that requires TLS. Omit or `false` for local MySQL without TLS. |
| `MYSQL_SSL_CA` | Filesystem path to CA bundle **inside** the process (e.g. in the Docker image at `/app/deploy/...`). |
| `MYSQL_SSL_CA_PEM` | Full PEM text of the CA (useful as a Render secret if you do not commit a file). |

## Alternative: MySQL as a Render private service

If you prefer everything on Render (no Aiven), see the older flow in Git history or use [Render’s MySQL Docker guide](https://render.com/docs/deploy-mysql). That path uses an internal hostname and does not expose MySQL to your laptop without SSH on Render.
